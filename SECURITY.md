# Security Policy

## Supported Versions

WinSwift is actively maintained on the `dev` and `master` branches. Only the latest version from the `master` branch is officially supported for security updates. 

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

If you discover any security-related issues (e.g., unintended privilege escalation, arbitrary code execution, or unintended system modifications outside of the stated scope of WinSwift), please do NOT open a public issue.

Instead, please privately report the vulnerability to the repository maintainers via GitHub Security Advisories or by emailing the project owner directly.

Please include the following information in your report:
- Type of issue (e.g. privilege escalation, code injection)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)

We take all security vulnerabilities seriously and will aim to resolve the issue as quickly as possible. We will try to acknowledge your report within 48 hours.
